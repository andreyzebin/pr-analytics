## Итоговый diff файла {file_path}:
Формат: -СТАРАЯ_СТРОКА +НОВАЯ_СТРОКА | код
  - строки с "-" = удалены из базовой ветки (колонка -old)
  - строки с "+" = добавлены в PR (колонка +new) — эти строки совпадают с исходным кодом выше
  - строки с " " = контекст без изменений (обе колонки)
```
{diff_content}
```

Чтобы найти якорь (строка {line}) в diff — ищи строку с +{line} в колонке +new.

## Инструкции:

1. Сначала проверь список коммитов после комментария.
   - Если файл НЕ фигурирует ни в одном коммите после комментария — замечание точно НЕ было учтено → NO.
   - Если файл был УДАЛЁН или ПЕРЕИМЕНОВАН — учти это при оценке.

2. Если файл менялся после комментария — найди строку якоря в diff и сравни замечание с изменениями.
   - Замечание просит удалить код → ищи строки "-" рядом с якорем
   - Замечание просит добавить код → ищи строки "+" после якоря
   - Замечание просит изменить код → ищи пару "-" (старое) / "+" (новое)

YES     — в diff видно исправление, явно соответствующее замечанию, И файл менялся после комментария
PARTIAL — изменения частично затрагивают проблему из замечания
NO      — файл не менялся после комментария, или изменения не связаны с замечанием
